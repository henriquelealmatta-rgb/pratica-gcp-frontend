Laboratório

Insights de PDFs com IA generativa

Documentos no Cloud Storage, processamento em VM sem IP público e modelo Gemini no Vertex AI — tráfego do usuário apenas na camada de borda.

Começar análise Ver arquitetura
Camadas 3 Borda · App · Dados/IA
Exposição pública 80/443 Somente frontend
Backend :5000 Tráfego interno VPC
IA Vertex Gemini + PDF via gs://
  • VPC Rede dedicada, subnets e regras explícitas.
  • Bucket PDFs versionáveis e fora da aplicação.
  • IAM SA na VM backend com papéis mínimos.
  • Proxy Nginx na borda encaminha /api.
  • 1

    Rede

    VPC, subnets e firewall: 80/443 na borda; backend só na rede interna.

  • 2

    Dados

    PDFs no bucket; leitura via SDK e contexto gs:// para o modelo.

  • 3

    IA

    Vertex AI com service account dedicada na VM de processamento.

Objetivos da prática

  • Isolar dados sensíveis da internet pública.
  • Demonstrar chamada privada ao modelo com contexto em GCS.
  • Reforçar princípios de menor privilégio na service account.

Serviços em destaque

Compute Engine Cloud Storage Vertex AI VPC Firewall IAM Nginx

Fluxo resumido

  1. Usuário acessa o frontend pela internet (HTTPS).
  2. API segue pela rede interna até o Flask na VM sem IP público.
  3. O backend lê metadados e PDF no bucket e chama o Gemini.
  4. A resposta volta ao navegador pelo mesmo caminho seguro.

Análise de documentos

Roteiro

  1. Bucket + PDFs de referência
  2. SA com Storage Viewer + Vertex User
  3. Firewall e teste de conectividade

Portas

  • Frontend 80 / 443
  • Backend 5000 (interno)

Variáveis

  • Projeto GOOGLE_CLOUD_PROJECT
  • Bucket GCS_BUCKET
  • Região VERTEX_LOCATION

Dica

Garanta que o PDF exista no bucket e que a região do Vertex coincida com a configurada no backend.
